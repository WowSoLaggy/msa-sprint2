# Миграция на Микросервис Бронирований

## План на ближайшие спринты

- Сервис бронирований выносится в отдельный микросервис.
- Подготовка регрессионных тестов микросервиса.
- Данные из боевой базы переносятся в новую базу микросервиса бронирований.
- Обращения к системе по-прежнему работают через монолит.
- Обращения к монолиту перенаправляются в новый микросервис используя gRPC.
- Новый микросервис по-прежнему зависит от остальных (ещё не вынесенных) частей монолита. Обращение к ним происходит через REST API.
- Подготовка (обновление) регрессионных тестов монолита с учётом вынесенного микросервиса.

## План на долгосрочную перспективу

- Вынос остальных частей монолита в отдельные микросервисы.
- Полная замена монолита на микросервисы.
- Обращение к микросервисам происходит через API Gateway, вместо монолита.
- Микросервисы взаимодействуют друг с другом через REST API при помощи Service Mesh.
